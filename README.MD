### 抖音电商ERP仓库（douyin_erp）介绍
该仓库是一个基于Python技术栈构建的**抖音电商轻量化ERP（企业资源计划）系统**，聚焦抖音店铺的日常运营数据管理、分析与可视化，适配中小商家的抖音电商运营需求，整体架构轻量化、易部署。

#### 一、仓库整体结构与核心目录
| 目录/文件          | 核心作用                                                                 |
|--------------------|--------------------------------------------------------------------------|
| `requirements.txt` | 项目依赖清单，定义了前后端所有核心依赖包及版本                          |
| `backend/`         | 后端核心模块，基于FastAPI构建API接口，处理业务逻辑与数据库交互           |
| ├─ `main.py`       | 后端入口文件，启动FastAPI服务、注册接口路由                             |
| ├─ `api/`          | 接口层，定义抖音店铺相关的API接口（如订单、商品、客户数据接口）          |
| ├─ `core/`         | 核心配置层，如数据库连接、抖音接口鉴权、全局常量等                       |
| ├─ `models/`       | 数据模型层，基于SQLAlchemy定义数据库表结构（如订单表、商品表、店铺表）    |
| ├─ `services/`     | 业务服务层，封装核心业务逻辑（如数据同步、订单处理、统计计算）           |
| `frontend/`        | 前端模块，基于Streamlit构建可视化Web界面，面向用户操作                  |
| ├─ `app.py`        | 前端入口文件，启动Streamlit应用，定义主页面布局                         |
| ├─ `pages/`        | 前端页面拆分，如“订单管理页”“数据报表页”“商品分析页”等                 |
| ├─ `utils/`        | 前端工具函数，如数据格式化、可视化图表封装、Excel导入导出辅助函数        |
| `data/`            | 数据存储目录                                                             |
| ├─ `douyin_shop.db`| SQLite数据库文件，存储抖音店铺的订单、商品、客户等核心数据（无需额外数据库服务） |

#### 二、核心技术栈（基于requirements.txt）
该项目全程基于Python生态，兼顾开发效率与轻量化部署，核心依赖如下：
| 依赖包              | 技术定位                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `fastapi==0.104.1`  | 后端高性能异步API框架，自动生成接口文档，适配抖音电商接口对接            |
| `uvicorn==0.24.0`   | ASGI服务器，用于运行FastAPI后端服务                                      |
| `streamlit==1.28.2` | 前端快速开发框架，无需前端基础即可构建数据可视化Web界面                |
| `sqlalchemy==2.0.23`| 数据库ORM框架，简化SQLite数据库的增删改查操作                            |
| `pandas==2.0.3`     | 核心数据处理库，用于抖音店铺数据的清洗、统计、分析                      |
| `openpyxl==3.1.2`   | Excel文件读写库，支持抖音店铺数据的Excel导入/导出                        |
| `plotly==5.17.0`    | 交互式可视化库，配合Streamlit实现订单、销量等数据的图表展示              |
| `python-multipart`  | 支持FastAPI处理文件上传（如Excel批量导入数据）                          |

#### 三、核心功能推测
基于架构与技术栈，该ERP系统核心能力包括：
1. **抖音店铺数据同步**：通过FastAPI接口对接抖音电商开放平台（或自定义爬虫），同步订单、商品、客户、销量等核心数据到SQLite数据库；
2. **数据管理操作**：前端页面支持订单/商品的增删改查、批量编辑、状态更新（如订单发货、商品上下架）；
3. **数据可视化分析**：基于Streamlit+Plotly实现销量趋势、订单分布、商品热销榜等交互式图表，直观展示店铺运营数据；
4. **数据导入导出**：支持Excel格式的订单/商品数据导入（批量录入）、导出（报表存档）；
5. **轻量化部署**：无需复杂的服务器配置，SQLite无需独立数据库服务，本地/轻量服务器即可运行，适配中小商家低成本使用。

#### 四、仓库特点
1. **轻量化**：无复杂前后端分离部署成本，Python一键安装依赖即可运行，适配小体量抖音店铺；
2. **开发高效**：FastAPI（自动接口文档）+ Streamlit（低代码前端）大幅降低开发/维护成本；
3. **数据驱动**：聚焦抖音电商核心数据的全流程管理，满足中小商家“数据看版+基础运营”的核心需求。
